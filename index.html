Show typical response playbooks (examples)

If credential exposure is found

  • Force reset on affected accounts (prioritized list)
  • Enable MFA everywhere possible
  • Revoke sessions / tokens where possible
  • Rotate API keys / secrets if exposure risk exists
  • Audit admin accounts and payment/fulfillment tools first

If an impersonation domain is found

  • Validate the domain and hosting details
  • File takedown requests (host/registrar/platform)
  • Update customer-facing warnings where needed
  • Improve email auth (SPF/DKIM/DMARC) if relevant
  • Monitor for re-registration and variants

FAQ

Straight answers. No hype. No vague promises.

Is this legal?

Yes. Obscurum uses lawful OSINT and threat intelligence methods. We do not hack systems, infiltrate private groups, or purchase stolen data.

Can you remove stolen data from the dark web?

No one can reliably erase criminal copies. What we do is reduce impact: resets, MFA enforcement, session/token revocation, and takedowns where feasible (impersonation/phishing).

What do you need from us?

Usually: domains, brand terms, email patterns (e.g., @brand.com), and a point of contact. Base packages typically do not require admin access.

What if you find nothing?

That’s still useful. You receive a report documenting what was checked and a prioritized prevention checklist to keep exposure low.

How fast are results?

Starter is typically 3–5 days. Exposure Sprint is typically ~7 days for most SMBs (complex brands may take longer).

Will you work with our IT provider/MSP?

Yes. We can hand off remediation steps and coordinate on takedown requests and control improvements.

Do you do penetration testing or internal security audits?

Not in the base Obscurum offering. This service is focused on external cybercrime visibility. If you need pentesting, we can refer you or discuss a separate scoped engagement.

What’s the biggest value of this service?

Speed. Visibility. Decision clarity. Most teams are blind to what’s happening outside their perimeter. Obscurum helps you respond before damage spreads.

Glossary (for non-security teams)
  • ATO (Account Takeover): attacker gains access to user accounts using stolen credentials.
  • Impersonation domain: a look-alike domain used to trick customers or staff.
  • Credential stuffing: automated login attempts using leaked credentials across many sites.
  • OSINT: Open Source Intelligence (lawful information gathering from open sources and permissible datasets).
